Atualização para o Angular 10

Este documento irá auxiliá-lo no uso do Angular 10.

Observação: Este documento é apenas para os que querem utilizar o Angular na versão 10. Ao longo do curso atualizaremos as dependências, mas caso queira atualizá-las agora, use este documento como guia.

Instalação do Boostrap

No momento não existe versão de testes para o Bootstrap, então iremos instalá-lo sem a utilização do @next: [code lang="shell"] npm install bootstrap --save [/code]

.angular-cli.json

Nesta versão não temos mais o arquivo .angular-cli.json, tendo este sido substituído pelo arquivo angular.json, que possui uma estrutura um pouco diferente, mas não entraremos em detalhes sobre sua estrutura agora.

Caminho relativo

Na aula é demonstrada a importação do arquivo CSS do Bootstrap utilizando o seguinte caminho:

[code] "../node_modules/bootstrap/dist/css/bootstrap.css" [/code]

Isso se deve ao fato de que no arquivo .angular-cli.json é atribuído o valor src para o atributo root, o que faz com que o caminho relativo seja baseado na pasta src do projeto. Nesta nova versão o valor padrão para o atributo root é uma string vazia, o que signfica que o caminho relativo se baseia no mesmo diretório do arquivo angular.json

Portanto, para que possamos importar o CSS do Bootstrap utilizamos o seguinte caminho:

[code] "node_modules/bootstrap/dist/css/bootstrap.css" [/code]

Veremos algumas das pequenas mudanças que podemos notar ao implementarmos o código da aula nesse nova versão.

Instalação do boostrap

No momento deste documento, não existe versão de testes para o bootstrap, então iremos instalá-lo sem a utilização do comando @next: npm install bootstrap --save

.angular-cli.json

Nessa versão, não temos mais este arquivo, tendo este sido substituído pelo arquivo angular.json que possui uma estrutura um pouco diferente, mas não entraremos em detalhes sobre sua estrutura agora.

Caminho relativo

Na aula é demonstrado a importação do arquivo CSS do bootstrap utilizando o caminho:

"../node_modules/bootstrap/dist/css/bootstrap.css"

Isso se deve ao fato de que no arquivo .angular-cli.json é atribuído o valor src para o atributo root, o que faz com que o caminho relativo seja baseado na pasta src do projeto.
